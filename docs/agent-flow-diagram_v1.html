Step-by-Step Reference

Step Name What Happens Who Acts Output
Step 0 Platform & Config Agent asks target platform (Web / Mobile / SAP / Salesforce / WinForms) and Tosca version. Sets correct TBox engine and selector technique priority. User Engine context locked in
Step 1 Discover & Inventory Reads all .cy.ts spec files, POM .ts classes, commands.ts, .feature files, and fixture files. Builds a complete inventory table for user confirmation. Agent Inventory list
Step 2 Pattern Analysis Maps each Cypress pattern to a Tosca primitive: it() → TestCase, POM getter → ModuleAttribute, Commands.add → ActionWord, Scenario → TestCase, cy.fixture → TestConfiguration. Agent Internal mapping model
Step 3 Flag Ambiguities Detects cy.intercept/stub (→ manual), if/else (→ Optional Step), cy.each (→ ForEach), dynamic selectors (→ parameterize), count assertions (→ Tosca Script). Presents resolution table to user. Agent + User resolves Resolution decisions
Step 4 Generate Tosca XML Writes TestCases-spec.xml, TestCases-feature.xml, Modules.xml, ActionWords.xml, TestConfigurations.xml into tosca-output/. Produces migration-report.md with accuracy rating per file (High/Medium/Low). Agent 5 XML files + report
Step 5 Review & Iterate User spot-checks one TestCase. Agent offers to refine selectors, rename elements, reconcile against existing Tosca ActionWord library, or migrate more files. User + Agent Refined XML, or next file batch
